`Tags` is used to render one or more linked tags.

## Examples

Tags align start (left) by default:

```jsx live
<Tags
  values={[
    { tag: 'css', path: '/tags/css/' },
    { tag: 'gatsby', path: '/tags/gatsby/' },
    { tag: 'nextjs', path: '/tags/nextjs/' },
    { tag: 'react', path: '/tags/react/' },
  ]}
  m={2}
/>
```

You can align center:

```jsx live
<Tags
  values={[
    { tag: 'css', path: '/tags/css/' },

    { tag: 'react', path: '/tags/react/' },
  ]}
  align="center"
  m={2}
/>
```

## System props

`Tags` supports [space props](https://styled-system.com/api#space) and
[color props](https://styled-system.com/api#color) from
[Styled System](https://styled-system.com/).

## Component props

| Name    | Type                       | Default        | Required | Description                                                                                                                                 |
| :------ | :------------------------- | :------------- | :------- | :------------------------------------------------------------------------------------------------------------------------------------------ |
| values  | Array of string            |                | âœ“        | Array of badge names.                                                                                                                       |
| gap     | Number                     | 2              |          | Gap between badges (from space scale).                                                                                                      |
| align   | `start`, `center` or `end` | `start`        |          | Alignment (responsive).                                                                                                                     |
| variant | String                     | `tags.primary` |          | Optional variant to customize the underlying [`Badge`](https://github.com/system-ui/theme-ui/blob/master/packages/components/src/Badge.js). |

## Variants

| Key         | Description                      |
| :---------- | :------------------------------- |
| `links.tag` | Customize links that wraps tags. |

If you do not provide a `variant` prop, tags are rendered with the defaults from
Theme UI's
[`Badge`](https://github.com/system-ui/theme-ui/blob/master/packages/components/src/Badge.js).
You can define your own variants under any key and provide that key to
`variant`. But it is recommended to define custom variants under the `tags` key.
If `Link` styling defined under `styles.a` interfers with tags, you can use key
`links.tag` to correct it.

In this example we make multiple customizations. We define variant
`tags.secondary` which changes colors and creates square corners and we override
default link styling with variant `links.tag`:

```jsx live
<ThemeProvider
  theme={{
    links: {
      tag: {
        textDecoration: 'none',
        '@media (hover: hover)': {
          '&:hover': {
            textDecoration: 'none',
          },
        },
      },
    },
    tags: {
      default: {
        borderRadius: 0,
      },
      secondary: {
        color: 'background',
        bg: 'secondary',
        variant: 'tags.default',
      },
    },
  }}
>
  <Tags
    values={[
      { tag: 'css', path: '/tags/css/' },
      { tag: 'gatsby', path: '/tags/gatsby/' },
      { tag: 'nextjs', path: '/tags/nextjs/' },
      { tag: 'react', path: '/tags/react/' },
    ]}
    variant="tags.secondary"
    m={2}
  />
</ThemeProvider>
```
